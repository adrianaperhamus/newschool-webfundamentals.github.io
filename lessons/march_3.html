GitHub Branches

We know that git takes snapshots of code called commits.

When we push our code to GitHub, we:
git push origin master

What is master?

Master is a branch. It's where you stage your commits.

You can also create and name your own branches to work on, which will commit your code not to your master project. This is how developers work together on the same project at once.

A gh-pages branch can be used to add code from other repos to your website. Let's walk through it.
